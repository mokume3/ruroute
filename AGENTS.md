## 一般ルール

- 必須：型よりもTypeScriptのインターフェースを使用してください。
- 必須：すべての型はグローバルスコープ内に保持してください。
- 必須：関数宣言よりもアロー関数を使用してください。
- 必須：デフォルトではコメントは追加しないでください。コメントを追加するのは、ユーザーが明示的に要求した場合、または「なぜ」が本当に明白でない場合（ブラウザの癖、プラットフォームのバグ、パフォーマンスのトレードオフ、脆弱な内部パッチ、直感に反する設計上の決定など）に限ります。コードの動作や、適切な名前の関数/変数が既に示している内容を繰り返すコメントは絶対に追加しないでください。迷った場合は、コメントは省略してください。
- 3行を超える説明的なコメントは、ユーザーに確認せずに削除しないでください。
- 必須：ファイル名はケバブケースを使用してください。
- 必須：変数名は分かりやすい名前を使用してください（省略形や1～2文字の名前は避けてください）。
  - 例: `.map()` では、`x` の代わりに `innerX` を使用できます。
  - 例: `moved` の代わりに `didPositionChange` を使用してください。
- 必須: 変数名をより正確で分かりやすいものにするために、頻繁に再評価およびリファクタリングしてください。
- 必須: 絶対に必要な場合を除き、型キャスト ("as") を使用しないでください。
- 必須: 未使用のコードを削除し、重複を避けてください。
- 必須: 常にコードベースを検索し、複数の解決策を検討した上で、最も_洗練された_解決策を実装してください。
- 必須: すべてのマジックナンバーは、`constants.ts` に `SCREAMING_SNAKE_CASE` と単位サフィックス (`_MS`、`_PX`) を付けて記述してください。
- 必須: 小さくて目的に特化したユーティリティ関数は、1 ファイルにつき 1 つのユーティリティ関数を `utils/` に配置してください。
- 必須: !! の代わりにブール値を使用してください。

## テスト

- 必須: テストを書いてから実装する（TDD: Red -> Green -> Refactor）。

コミット前に必ず以下のコマンドでチェックを実行してください。

```bash
pnpm test # テストを実行
pnpm check # biomeチェックを実行
pnpm typecheck # 型チェックを実行
```

## 開発手順

これはtsdownでのライブラリパッケージプロジェクトです。  
外部サービス（データベース、Dockerなど）は不要です。

### テスト前にビルド

`pnpm build` は `pnpm test` または `pnpm check` を実行する前に完了している必要があります。ソースファイルを変更した後は、必ずテストを実行する前にビルドしてください。

### 主要コマンド一覧

コマンド一覧については、ルートディレクトリの `package.json` スクリプトを参照してください。クイックリファレンス：

- **インストール**: `pnpm install`
- **ビルド**: `pnpm build`
- **開発監視**: `pnpm dev` - パッケージを監視
- **テスト**: `pnpm test` - テストを実行 (Vitest)
- **型チェック**: `pnpm typecheck` - tsc
- **Lint/フォーマット**: `pnpm check` - biomeチェック

## パフォーマンス

- 必須: `for`/`while`ループと文字列連結のホットパスは保持してください。`map`/`filter`/`reduce`/`join`、または配列スプレッド（`push(...arr)`）は、呼び出しごとのコードでは絶対に使用しないでください。
- 必須: `Object.entries`/`Object.keys`/spread/`Object.fromEntries`は、初期化時（設定構築時）のみに制限してください。呼び出しごとのコードでは絶対に使用しないでください。
- 必須: LRU/ルックアップマップには `Object.create(null)` を使用し、頻繁に使用されるパスでは `Map` に「近代化」しないこと。
- 必須: 正規表現、バリデーター、定数はモジュールスコープに上げ、クラスごとの関数内で `RegExp` やクロージャを構築しないこと。
- 必須: `delete` は使用せず、代わりに上書き/再利用すること。遅延初期化を維持する（`initTailwindMerge` を自己パッチする）。
